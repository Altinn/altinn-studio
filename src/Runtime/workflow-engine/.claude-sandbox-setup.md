# Sandbox Environment Setup for workflow-engine

## 1. Mount the repository

The sandbox needs access to at least `src/Runtime/` (not just `src/Runtime/workflow-engine/`)
because the API project depends on `../../../common/Altinn.Studio.Runtime.Common.csproj`.

Ideally mount the full repo or at minimum `src/Runtime/`.

## 2. Install .NET 10 SDK

The project targets `net10.0`. Ubuntu 25.10 (the default sandbox OS) has it in the standard repos:

```bash
sudo apt-get update -qq && sudo apt-get install -y -qq dotnet-sdk-10.0
```

Verify: `dotnet --version` → should show `10.0.x`

The project's `global.json` requires SDK `10.0.101` with `rollForward: latestFeature`, so any `10.0.1xx` works.

## 3. Navigate to the working directory

```bash
cd <repo-root>/src/Runtime/workflow-engine
```

## 4. Clean stale build artifacts

The sandbox gets stale `obj/` dirs from the host mount. Clean them before the first build:

```bash
find . -name "obj" -type d -exec rm -rf {} + 2>/dev/null
find . -name "bin" -type d -exec rm -rf {} + 2>/dev/null
dotnet nuget locals all --clear
```

## 5. Build

```bash
dotnet build tests/WorkflowEngine.EndToEndTests/WorkflowEngine.EndToEndTests.csproj
```

## 6. Run tests

Tests require Docker (for Testcontainers/PostgreSQL). Verify Docker is available first:

```bash
docker info
```

Then:

```bash
dotnet test tests/WorkflowEngine.EndToEndTests/WorkflowEngine.EndToEndTests.csproj
```
